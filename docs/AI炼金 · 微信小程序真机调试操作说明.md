---

# AI炼金 · 微信小程序真机调试操作说明

## 1. 架构与核心原理

| 组件         | 运行位置              | 说明                                               |
| ------------ | --------------------- | -------------------------------------------------- |
| 小程序前端   | 微信开发者工具 + 手机 | 工具目录为 `frontend`，编译产物在 `frontend/dist/` |
| FastAPI 后端 | 本机终端，端口 `8000` | 命令：`npm run dev:backend`（监听 `0.0.0.0:8000`） |
| MySQL        | 本机 `127.0.0.1:3306` | 仅后端访问，手机无需直连                           |

**真机与模拟器的最大区别：**

- 模拟器里 `http://127.0.0.1:8000` 指向你的电脑，可以正常访问后端。
- 真机上 `127.0.0.1` 指向手机自身，**无法访问电脑上的后端**。
- 真机调试时，小程序必须把 API 地址改成你电脑的**局域网 IP**，例如 `http://192.168.1.100:8000`。

当前开发配置（`frontend/config/dev.ts`）默认是：

```ts
API_BASE_URL: '"http://127.0.0.1:8000"'
```

因此：**不做地址调整，真机几乎一定会出现「请求失败 / request:fail」**。

## 2. 前置条件清单

开始之前确认：

- [ ] 微信开发者工具已安装，并已用**同一 AppID** 打开项目（当前为 `wxae9e3547ab934143`）
- [ ] 你的微信账号是该小程序的**开发者或体验成员**（公众平台 → 成员管理）
- [ ] 手机与电脑连接**同一 Wi‑Fi**（避免手机用 4G、电脑用 Wi‑Fi）
- [ ] 本机 MySQL 已启动，数据库 `ai_alchemy` 可用
- [ ] 后端 `.env` 中 `WECHAT_APP_ID` / `WECHAT_APP_SECRET` 已配置（你当前为真实微信登录）
- [ ] Windows 防火墙允许本机 **8000 端口** 被局域网访问（见下文「常见问题」）

## 3. 一次性网络配置（真机调试必做）

### 3.1 查询电脑局域网 IP

在 PowerShell 中执行：

```powershell
ipconfig
```

找到当前 Wi‑Fi 适配器的 **IPv4 地址**，例如 `192.168.1.100`（下文用 `<你的IP>` 代替）。

### 3.2 修改前端 API 地址（真机调试前）

编辑 `frontend/config/dev.ts`，将 `API_BASE_URL` 改为：

```ts
API_BASE_URL: '"http://<你的IP>:8000"'
```

示例：

```ts
API_BASE_URL: '"http://192.168.1.100:8000"'
```

> 真机调试结束后，若只在模拟器开发，可改回 `127.0.0.1`。

### 3.3 修改后端公开 URL（头像等资源）

若会测试**头像上传/显示**，还需在 `server/.env` 中设置：

```env
PUBLIC_BASE_URL=http://<你的IP>:8000
```

否则后端返回的头像链接仍是 `http://127.0.0.1:8000/...`，真机上图片无法加载。

### 3.4 重新编译前端

在 `frontend` 目录执行：

```powershell
cd d:\Code\ai-learn-go\frontend
npm run dev:weapp
```

保持 watch 模式运行，保存后会自动编译到 `dist/`。

## 4. 日常启动顺序

建议按以下顺序操作：

```
1. 启动 MySQL
2. 启动后端
3. 启动前端编译（watch）
4. 打开微信开发者工具
5. 真机调试 / 预览
```

### 4.1 启动后端

在项目根目录：

```powershell
cd d:\Code\ai-learn-go
npm run dev:backend
```

等价于：

```powershell
cd server
.venv\Scripts\uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

验证：在电脑浏览器打开：

```
http://<你的IP>:8000/api/v1/health
```

应返回 `{"status":"ok"}`。  
若本机 `127.0.0.1` 能通、`<你的IP>` 不通，多半是防火墙问题。

### 4.2 打开微信开发者工具

1. **导入项目** → 目录选择：`d:\Code\ai-learn-go\frontend`（不是 `dist`，工具会根据 `project.config.json` 的 `miniprogramRoot: "dist/"` 读取编译产物）
2. AppID 选择与线上一致的 `wxae9e3547ab934143`
3. 确认编译成功，模拟器能正常打开首页

### 4.3 开发者工具关键设置

路径：**详情 → 本地设置**

建议勾选：

- **不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书**

你项目里 `project.config.json` 已设 `"urlCheck": false`，但仍建议在界面中确认已勾选。

> 说明：真机调试/预览属于开发态，可用 HTTP + 局域网 IP。正式上线必须在公众平台配置 **request 合法域名（HTTPS）**。

## 5. 真机调试 vs 预览

### 方式 A：真机调试（推荐，可排查问题）

1. 微信开发者工具顶部点击 **「真机调试」**
2. 用手机微信扫描弹出的二维码
3. 手机打开小程序，并进入**调试模式**（可查看 vConsole、网络请求等）
4. 在开发者工具的 **「真机调试」面板** 查看 Console、Network

适用：接口报错、登录失败、页面白屏等需要看日志的场景。

### 方式 B：预览（快速体验）

1. 点击 **「预览」**
2. 手机微信扫码
3. 在手机上体验功能，调试能力较弱

适用：快速验证 UI 和主流程，不需要深入排错。

---

## 6. 推荐测试流程

真机连上后，建议按顺序验证：

| 步骤 | 操作                     | 验证点                                           |
| ---- | ------------------------ | ------------------------------------------------ |
| 1    | 打开「我的」页，点击登录 | 微信 `wx.login` + 后端 `/api/v1/auth/login` 成功 |
| 2    | 首页输入关键词，发起出题 | 请求打到 `http://<你的IP>:8000`，轮询任务正常    |
| 3    | 完成答题、查看报告       | 报告接口、长轮询/超时是否正常                    |
| 4    | 上传头像（如有）         | 图片 URL 使用 `PUBLIC_BASE_URL`，非 127.0.0.1    |
| 5    | 分享/海报（如有）        | 保存相册等需真机权限的功能                       |

在真机调试面板的 **Network** 中，确认请求 Host 为 `<你的IP>:8000`，而不是 `127.0.0.1`。

## 7. 常见问题排查

### 7.1 请求失败 / request:fail

| 可能原因                   | 处理                                                 |
| -------------------------- | ---------------------------------------------------- |
| API 仍为 `127.0.0.1`       | 按 §3.2 改 `dev.ts` 并重新编译                       |
| 手机与电脑不在同一网段     | 统一 Wi‑Fi；关闭手机流量                             |
| 防火墙拦截 8000            | Windows 防火墙 → 入站规则 → 允许 TCP 8000            |
| 后端未启动或未监听 0.0.0.0 | 确认 `npm run dev:backend` 在跑，且 `--host 0.0.0.0` |
| 改 IP 后未重新编译         | 重启 `npm run dev:weapp`，工具中点「编译」           |

### 7.2 登录失败

- 确认 `.env` 中 `WECHAT_APP_ID`、`WECHAT_APP_SECRET` 与小程序 AppID 一致
- 确认扫码微信是**该小程序开发者/体验者**
- 真机调试面板查看 `/api/v1/auth/login` 返回的错误信息

### 7.3 头像/静态资源不显示

- 检查 `PUBLIC_BASE_URL` 是否为 `http://<你的IP>:8000`
- 浏览器直接访问返回的头像 URL 是否能打开

### 7.4 扫码后无法打开 / 提示无权限

- 公众平台 → **成员管理** → 将你的微信加为开发者或体验成员
- 确认 AppID 与项目一致

### 7.5 路由器「AP 隔离 / 访客网络」

部分路由器会禁止 Wi‑Fi 设备互访。若 `<你的IP>:8000` 在手机浏览器也打不开，检查路由器是否开启 **AP Isolation / 访客网络隔离**。

### 7.6 不在同一局域网时的替代方案（进阶）

若无法保证同 Wi‑Fi（例如公司网络限制），可考虑：

- 用 **ngrok / frp / Cloudflare Tunnel** 将本地 `8000` 暴露为公网 HTTPS 地址
- 或将后端部署到云服务器（`frontend/config/prod.ts` 中的生产地址）

这属于进阶方案，日常家庭/办公室同 Wi‑Fi 调试通常不需要。

## 10. 小结

真机调试的**三个关键点**：

1. **API 地址必须用局域网 IP**，不能用 `127.0.0.1`
2. **手机与电脑同一 Wi‑Fi**，且防火墙放行 8000 端口
3. **开发者工具开启「不校验合法域名」**，并用「真机调试」扫码测试

按 §3 改好地址并重新编译后，再按 §4 → §5 操作，即可在真机上完整测试依赖本地后端的各项功能。
